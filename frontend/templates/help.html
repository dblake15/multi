How to use Launch - Quickstart Guide

You've gotten to the web-page, now what? This tutorial will walk you through the basics of how to use Launch.

  1. When you first navigate to Launch, you are met with a screen that looks like this:
  2. Enter in your GitHub username, or the username under which the project you are trying to deploy has been stored.
  3. Click the 'Begin' button, and the next page will auto-populate with your repositories.
  4. The next page should look like this:
  5. Select your project's repository from the drop-down, and make sure the 'Create' option is selected, then press the 'Go' button
  6. After a few seconds, you should see a message saying your application has been deployed
  7. Make a note of the port it is running on, and navigate to the IP address of your cluster, followed by the port you noted earlier.
  8. For example, if your project is running on port 32123, and your cluster IP address is 192.168.90.100, your project lives at http://192.168.90.100:32123

UI Walkthrough

At its core, the Launch UI is supposed to be simple.

You'll notice that you only need to fill in one text box, and the rest is either drop down or radio buttons.

That being said, there is much to be gained from a deep understanding of the user interface and all of its parts.

  • GitHub Username:

    This is where you will enter your GitHub username so that Launch can pull information about all of your public repositories

  • Repository:

    This dropdown menu is an autopopulated list of your public repositories. Simply select the correct repo from the list and you'll be on your way!

  • Create/Delete:

    These radio buttons tell Launch whether you want to Create a new Project or Delete it based on the repository name.

  • Deployments associated with this username:

    When you have deployments asociated with your GitHub username, they will appear here as a list of links.

    These links simply take the current IP address and append the correct port to it, so your mileage with these links may vary.

    While this method is convenient, and generally accurate, it may serve you best to also keep your own records of where your projects live.

Create a Launch-Friendly Application

One might find importance in the knowledge of how to create an app that Launch is capable of deploying.

Launch is currently only capable of deploying static web applications, although work is being done to allow for dynamic applications.

However, if you have a static web application that you would like to deploy with Launch, all you need to do is:

  1. Create a Dockerfile for your project
  2. Upload your project to a public repository on GitHub
  3. That's it!
